# Premium Perception Audit — All Pages (2026-04-24)

Запускаю премиум-аудит. Объект оценки — восприятие ценности, а не дизайн. Без оператора ПОДТВЕРЖДАЮ работаю в режиме сенсора. Поехали.

## Scope
- Home + shared sections/components.
- Blog list + article page.
- FAQ and service/support pages loaded via router lazy boundaries.
- Global style tokens, transitions, focus states, loading states.

---

## 🔴 CRITICAL (блокеры доверия)

### C-01 — Непредсказуемые состояния загрузки между страницами
**Симптом:** Для lazy-роутов и загрузки статей используется минимальный текстовый fallback `Загрузка...` без визуального каркаса и без сохранения ритма экрана.  
**Диагноз:** В дорогом интерфейсе ожидание должно быть «контролируемым». Текстовый спиннер-заменитель выглядит как временная заглушка, а не как законченный продукт.  
**Рецепт (Figma/DevTools):**
1. Ввести единый premium-loading pattern (skeleton + тихая пульсация 1.4–1.8s).
2. Зафиксировать высоты hero/content-блоков в состоянии загрузки, чтобы не ломалась композиция.
3. Использовать один компонент для всех страниц (`RouteSkeleton`, `ArticleSkeleton`, `SectionSkeleton`).
**Прогноз:** Меньше тревожности при переходах, выше ощущение инженерной зрелости.  
**Safe Mode:** Добавить только визуальный fallback-слой, не меняя тексты, маршруты и данные.

### C-02 — Перегруженные анимационные акценты в ключевых CTA
**Симптом:** На кнопках и карточках одновременно применяются градиенты, блеск-шейн, scale и длинные переходы.  
**Диагноз:** Премиум-сегмент считывает это как «маркетинговый шум». Дорогой интерфейс показывает уверенность, а не пытается «продавить» внимание.  
**Рецепт (Figma/DevTools):**
1. Оставить один ведущий акцент на компонент (например, только depth + color shift).
2. Убрать избыточные шейны с длительностью 800–1000ms на базовых CTA.
3. Нормировать duration: реакция 90–120ms, завершение 260–360ms, easing `cubic-bezier(.22,1,.36,1)`.
**Прогноз:** Интерфейс воспринимается как спокойный и дорогой, CTA становятся более «механическими», а не декоративными.  
**Safe Mode:** Вынести новые motion-токены в CSS переменные, включить по feature-flag классу.

## 🟡 HIGH IMPACT (усиление ценности)

### H-01 — Нестабильная система интерактивных состояний (:hover/:focus/:active)
**Симптом:** Часть интерактивов имеет богатые состояния, часть — почти безобратную реакцию (особенно текстовые кнопки/ссылки).  
**Диагноз:** Пользователь теряет «чувство сцепления» с интерфейсом: не всегда понятно, где action, а где статичный элемент.  
**Рецепт (Figma/DevTools):**
1. Ввести матрицу состояния для всех control-типов (link, button, chip, card).
2. Добавить единый `focus-visible` контур (цвет ring из бренда + мягкая внешняя тень).
3. Убедиться, что `active` ощущается «продавливанием» (translateY 1px + shadow reduction), а не только сменой цвета.
**Прогноз:** Больше тактильности и контроля, особенно на мобильных и клавиатурных сценариях.  
**Safe Mode:** Начать с shared UI (`Button`, ссылочные утилиты), затем каскадно подключать к страницам.

### H-02 — Вертикальный ритм и оптическая иерархия не унифицированы между страницами
**Симптом:** Некоторые экраны (например, блог-статья vs список) имеют разный «шаг дыхания»: где-то плотный стек, где-то слишком воздушно.  
**Диагноз:** Продукт выглядит как набор хороших секций, а не как единая система высокого класса.  
**Рецепт (Figma/DevTools):**
1. Ввести spacing scale для page rhythm: 8/12/16/24/32/48/64.
2. Закрепить шаблон: `PageTop`, `SectionGap`, `CardInnerGap`, `ReadingWidth`.
3. На мобиле увеличить минимальный межблочный gap в длинных текстовых секциях.
**Прогноз:** Структура считывается за 1–2 секунды даже без чтения, растет «нативность» продукта.  
**Safe Mode:** Рефактор только через токены spacing, без перестройки DOM и контента.

### H-03 — Единый слой теней и глубины пока не сформирован
**Симптом:** В разных блоках используются разные по характеру тени (иногда агрессивные, иногда почти отсутствуют).  
**Диагноз:** Разнородная глубина = ощущение «собрано из разных систем». Это прямой сигнал самодельности.  
**Рецепт (Figma/DevTools):**
1. Принять 3 уровня elevation (surface / raised / floating).
2. Перейти на многослойные бренд-нейтральные тени (не черные плоские пятна).
3. Поддержать ту же систему для dark background (с учетом цветовой подсветки).
**Прогноз:** Карточки и модальные слои станут «материальными», возрастёт воспринимаемое качество.  
**Safe Mode:** Вынести тени в utility-классы и заменить точечно в shared компонентах.

## ⚪ POLISH (эстетическая гигиена)

### P-01 — Несогласованная «скорость интерфейса»
**Симптом:** У разных эффектов разброс длительности и характера движения слишком велик.  
**Диагноз:** Даже красивые эффекты при разной физике дают ощущение неполированной сборки.  
**Рецепт:** Ввести motion scale (`fast`, `base`, `slow`) и использовать только её.  
**Прогноз:** Единое «дыхание» продукта.  
**Safe Mode:** Добавить токены и запретить raw-duration в новых компонентах.

### P-02 — Местами слабая читаемость вторичного текста на темном фоне
**Симптом:** Часть muted-текста может быть «ок на тесте», но утомляет в длинном чтении.  
**Диагноз:** Премиум ≠ низкий контраст. Премиум = комфорт при длительном скролле.  
**Рецепт:** Поднять контраст вторичного текста на 6–10% в длинных текстах, не трогая брендовые акценты.  
**Прогноз:** Ниже fatigue, выше ощущение «чистоты воздуха».  
**Safe Mode:** Изменять только текстовые utility-классы для контентных блоков.

### P-03 — Не везде явно различимы кликабельные зоны у карточек
**Симптом:** Некоторые карточки кликабельны целиком, но это считывается только после hover.  
**Диагноз:** До наведения пользователь тратит когнитивный ресурс на распознавание affordance.  
**Рецепт:** Добавить базовый интерактивный сигнал в idle-state (курсор + subtle outline/gradient edge).  
**Прогноз:** Быстрее принятие решения и меньше микропауз «можно кликнуть или нет».  
**Safe Mode:** Только визуальный affordance, без изменения логики клика.
