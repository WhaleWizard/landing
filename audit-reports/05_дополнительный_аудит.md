# Файл: 05_дополнительный_аудит.md

## 1. Что могло быть пропущено в первом аудите и добавлено после уточнений
- 🟢 Уточнён production-контекст Cloudflare: D1 `landing-prod`, R2 uploads bucket, KV для Meta diagnostics/idempotency/nonce, direct analytics runtime, canonical `SITE_URL`, fresh articles requirement и server-side outbox attempts.
- 🟢 Уточнено назначение сайта: коммерческий performance marketing сайт Whale Wzrd с услугами Google Ads/Meta Ads/консультаций, блогом, кейсами, глоссарием, калькуляторами, формами лидогенерации, thank-you страницей, legal pages и админкой статей.
- 🟢 Уточнено, что auto-enable cookies при неизвестном регионе — намеренное продуктово-юридическое решение, а не случайный дефект. В отчётах оно переоценено как policy risk/monitoring item.
- 🟡 Что всё ещё нельзя подтвердить без внешних данных: реальные Cloudflare response headers в production, содержимое GTM container, Search Console/Webmaster ошибки, Yandex Webvisor записи на мобильных, CRM/Google Sheets фактическое получение лидов, R2 cache headers, Cloudflare WAF/rate-limit rules, actual D1 freshness SLA, логи Meta CAPI diagnostics.

## 2. Service Workers и PWA
- 🟢 В аудированных файлах service worker registration не найден; риск stale cache низкий.
- 🟡 Нет PWA offline fallback. Для landing это не критично, но offline submit queue для лидов полезна.
- 🟡 Если добавить PWA, обязательно: versioned precache, network-first для API, cache allowlist, clear old caches.

## 3. Состояния загрузки
- 🟢 `RouteSkeleton` используется как Suspense fallback.
- 🟢 ContactForm показывает `Loader2` и disabled submit.
- 🟡 Нет skeleton для статей при API latency на slow 3G во всех компонентах (нужно проверить BlogPage).
- 🟡 Optimistic UI для лидов не нужен, но нужна очередь/retry.

## 4. Ошибки 404/500/обрыв соединения
- 🟢 NotFound route есть.
- 🟢 RouteErrorBoundary обрабатывает failed dynamic import.
- 🟡 Cloudflare SPA fallback может отдавать 200 для неизвестных путей; нужна проверка production status.
- 🟡 API errors в формах показываются alert; заменить на inline state.

## 5. Мультиязычность и локализация
- 🟢 Основной язык `ru` задан.
- 🟡 В SEO areaServed заявляет RU/US/AE/TR/EU, но нет полноценных локализованных версий и hreflang.
- 🟡 Валюта/форматы телефонов смешаны ($ budget, ru copy, US default +1). Для международного mobile UX лучше определять locale/currency.

## 6. Реальные устройства и сети
Необходимо проверить:
1. Lighthouse mobile: Slow 4G, 4× CPU.
2. WebPageTest Moto G4/Android Chrome.
3. iOS Safari real device: keyboard resize, scroll lock modals.
4. Android WebView in Telegram/Instagram browser: cookies, localStorage, form submit.
5. 3G loss during lead submit.

## 7. Voice search / featured snippets
- 🟡 FAQ page есть, но нужно добавить FAQPage JSON-LD server-side.
- 🟡 Заголовки должны отвечать на вопросы кратко: "Сколько стоит настройка Google Ads?", "Как быстро запускается реклама?".
- 🟡 Добавить блоки definition/list/table для featured snippets.

## 8. Юридические риски
- 🟢 Страницы privacy-policy, offer, cookie-policy есть.
- 🟡 Нужно проверить соответствие фактическим third-party: Google, Yandex, Meta, TikTok, JSONBin, ipwho.is, Cloudflare, R2 public host, D1/KV processing, Google Apps Script/webhooks.
- 🟡 Для FZ-152 нужны оператор, цели, сроки, трансграничная передача, права субъекта, contact.
- 🟡 Для GDPR нужны lawful basis, DPA/SCC, consent withdrawal, data retention.

## 9. Утечки памяти SPA
- 🟢 Effects в ContactForm cleanup active flag.
- 🟢 RouteErrorBoundary чистит timeout.
- 🟡 Нужно профилировать components with animations/intersection observers/carousels.
- 🟡 Long-lived SPA analytics event listeners должны иметь cleanup.

## 10. Мониторинг и CI/CD
- 🟡 Sentry/Rollbar не обнаружены в ключевых файлах. ErrorBoundary без remote reporting снижает observability.
- 🟡 Lighthouse CI не обнаружен в package scripts.
- 🟡 Нет typecheck/lint scripts; `test` фактически равен build.

Решение scripts:
```json
{
  "scripts": {
    "typecheck": "tsc --noEmit",
    "lint": "eslint .",
    "lhci": "lhci autorun",
    "audit:deps": "pnpm audit --prod"
  }
}
```

## 11. Мобильные артефакты
- 🟠 Возможна потеря лида на mobile offline/network switch.
- 🟡 Consent banner/modals нужно проверить на 320px: не перекрывают ли CTA полностью, есть ли scroll lock.
- 🟡 Touch events: hover отключён локально, но global hover styles могут залипать на iOS.
- 🟡 Mobile keyboard: поля формы ниже fold, нужен `scrollIntoView` на focus/error.
- 🟡 Hidden desktop-only content: нужно сравнить DOM/text на mobile vs desktop screenshots, чтобы mobile-first indexing не терял контент.

## 12. Дополнительные критичные пробелы
- 🟠 Нет field RUM. Добавить `web-vitals` и отправлять LCP/INP/CLS с device/network info.
- 🟠 Нет content freshness alert для JSONBin fallback; build может пройти со старым local data.
- 🟡 Нет documented dataLayer schema. События могут расползтись между GA/YM/Meta.
- 🟡 Нет automated visual regression на generated static pages.
